# Plans

## Purpose

This directory contains project plans, timelines, milestones, and high-level schedules for organizing work across corporate finance projects.

## What Belongs Here

- Project plans and roadmaps
- Timeline and schedule documents
- Milestone definitions and completion criteria
- Workstream breakdowns
- Resource and capacity plans
- Critical path analyses
- Phase gates and stage gates

## Naming Conventions

- **Project plans:** `PLAN_[ProjectName]_[Phase].md`
  - Example: `PLAN_DCFValuation_Phase1.md`
- **Timelines:** `TIMELINE_[ProjectName]_[DateRange].xlsx`
  - Example: `TIMELINE_CashFlowModel_2026-05-01_to_2026-06-30.xlsx`
- **Milestones:** `MILESTONES_[ProjectName].md`
  - Example: `MILESTONES_CapitalBudgeting.md`
- **Roadmaps:** `ROADMAP_[Initiative]_v[Version].md`
  - Example: `ROADMAP_ModelingFramework_v1.md`

## Getting Started

1. Define milestones with clear completion criteria
2. Include start and end dates for each phase
3. Link to related decision documents and specifications
4. Update plans as conditions change
5. Archive superseded plans with notes on updates

## Guidelines

- Keep plans realistic and achievable
- Identify dependencies between workstreams
- Include contingency and buffer time
- Assign owners and stakeholders
- Review and update plans regularly
- Link phase gates to decision milestones
