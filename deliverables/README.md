# Deliverables

## Purpose

This directory contains final, presentation-ready outputs including reports, memos, analyses, and other formal deliverables for corporate finance projects and coursework.

## What Belongs Here

- Final presentation decks and executive summaries
- Formal written reports and decision memos
- Published analyses and findings
- Client-ready documentation
- Final model outputs and executive dashboards
- Signed-off deliverables

## Naming Conventions

- **Reports:** `[ProjectName]_Report_[Type]_v[Version]_[Date].pdf`
  - Example: `CapitalBudgeting_Report_Analysis_v2_2026-05-15.pdf`
- **Memos:** `MEMO_[Subject]_[Date].docx`
  - Example: `MEMO_FundingDecision_2026-05-15.docx`
- **Presentations:** `[ProjectName]_Presentation_[Date].pptx`
  - Example: `DCF_Presentation_2026-05-15.pptx`
- **Dashboards:** `Dashboard_[MetricCategory]_[Date].xlsx`
  - Example: `Dashboard_FinancialMetrics_2026-05-15.xlsx`

## Getting Started

1. Ensure all deliverables are complete and validated before placing here
2. Include version numbers and dates in file names
3. Keep a change log for each major deliverable
4. Update this README when new deliverable categories are added
5. Archive older versions when new versions are finalized

## Guidelines

- All deliverables must be review-ready or approved
- Include executive summaries for complex documents
- Maintain consistency in formatting and branding
- Store original source files in `/docs/` or `/models/`
- Use this directory as the single source of truth for final outputs
- Keep documentation of stakeholder sign-offs
